Introducción y presentación de DB Browser

Al abrir DB Browser nos aparecerá el menú principal. En él encontramos varias pestañas con los nombres "Estructura", "Hoja de datos", "Editar programas" y "Ejecutar SQL".

Los que a nosotros más nos interesan son el primero y el último (aunque "Hoja de datos" es útil para ver cómo son las tablas que tengamos sin tener que escribir código).

En la pestaña de "Estructura" encontramos las tablas que hemos creado. Ahora mismo no nos debería aparecer ninguna.

Lo primero que se tiene que hacer es crear una base de datos, pulsando el botón "Nueva Base de datos". Te pedirá guardarla en tu ordenador (asegúrate de que tenga la extensión .db).

Cuando la hayamos creado y guardado, nos va a aparecer este menu. Este menu sirve para crear tablas de una manera intuitiva sin tener que usar codigo, lo genera solo abajo.

Aquí podemos darle nombre a la tabla y añadir las columnas que queramos.

Nosotros vamos a ignorar este menu y vamos a crear las tablas con el propio codigo, así que lo podemos cerrar.

Si nos moveremos a la pestaña "Ejecutar SQL" vemos tres grandes bloques. El de arriba es donde se escribe el código, el del medio es donde se verá el resultado del código, si lo hay; y el último nos dirá si ha habido algun error al ejecutar.

Crear una tabla

Empezamos con lo divertido del curso: el código.

Lo primero que vamos a ver es como crear tablas.

SQL
 CREATE TABLE 'usuarios' (
	'nombre'	TEXT,
	'apellido'	TEXT,
	'edad'	INTEGER
)

Con este código hemos creado una tabla llamada "usuarios" con tres columnas: "nombre", "apellido" y "edad".

Como vemos, el código es bastante intuitivo. En su sintaxis vemos primer el nombre de la tabla, seguido de las columnas que queremos añadir, entre parentesis y separadas por comas. Para las columnas primero ponemos el nombre de ésta entre '' (o "") seguido de su formato. El formato indica que tipo de valor va a haber en la celda de cada columna. Por ejemplo, el nombre será texto así que su formato será TEXT. Para la edad, será un número entero, así que su formato será INTEGER.

A parte de estos dos formatos hay muchos más. Los más usados son DATE, para fechas; REAL, para valores con decimales, limitados a 64 bytes; NUMERIC, para valores con decimales sin limite de bytes (como pi); o BLOB, para el formati binario.

Insertar datos

Ahora que ya tenemos nuestra tabla, vamos a llenarla con datos. Para eso usamos el comando INSERT INTO.

SQL
INSERT INTO usuarios ('nombre', 'apellido', 'edad') 
VALUES ('Juan', 'Pérez', 30),
		('María', 'Gómez', 25),
		('Carlos', 'López', 35);

Consultar datos

Ahora que ya tenemos datos en nuestra tabla, vamos a aprender a consultarlos. Para eso usamos el comando SELECT.

SQL
SELECT nombre, apellido FROM usuarios;

Con este código hemos seleccionado las columnas "nombre" y "apellido" de la tabla "usuarios". El resultado es una tabla con los nombres y apellidos de los usuarios, pero sin la edad.

Si queremos seleccionar todas las columnas, podemos usar el asterisco * en lugar de escribir cada columna. Por ejemplo:

SQL
SELECT * FROM usuarios;

Esto nos traerá todas las columnas de la tabla "usuarios".

Identificadores

En SQL, los identificadores sirven para poder identificar cada fila con un número único. Son muy útiles para tablas que tenga filas muy parecidas. Es como el DNI de cada fila. Es importante seguir ciertas reglas al nombrar estos identificadores para evitar errores y confusiones.

Algunas de las reglas más importantes son:

  • Pueden contener letras, números y guiones bajos.
  • No pueden contener espacios ni caracteres especiales (como @, #, $, etc.).
  • No pueden ser palabras reservadas de SQL (como SELECT, FROM, WHERE, etc.).

Los identificadores no pueden tener celdas vacias así que no podemos insertar ahora un identificador a nuestra tabla, ya que las filas ya creadas apareceria su correspondiente celda vacia. Entonces primero tenemos que eliminar la tabla y volver a crearla para añadir los valores a este identificador:

SQL
DROP TABLE usuarios;
CREATE TABLE 'usuarios' (
	'DNI' INTEGER,
	'nombre'	TEXT,
	'apellido'	TEXT,
	'edad'	INTEGER,
	PRIMARY KEY ('DNI' AUTOINCREMENT)
);

Con este código hemos eliminado la tabla "usuarios" y la hemos vuelto a crear añadiendo una nueva columna llamada "DNI" que será nuestro identificador. Además, hemos indicado que esta columna será la clave primaria (PRIMARY KEY) y que se autoincrementará (AUTOINCREMENT) cada vez que añadamos una nueva fila. De esta manera, cada fila tendrá un número único en la columna "DNI".

Ahora que ya tenemos nuestro identificador, podemos insertar los datos de nuevo. Esta vez, como el identificador se autoincrementa, no necesitamos especificarlo en el código de inserción:

SQL
INSERT INTO usuarios ('nombre', 'apellido', 'edad') 
VALUES ('Juan', 'Pérez', 30),
    ('María', 'Gómez', 25),
    ('Carlos', 'López', 35);
SELECT * FROM usuarios;

Como vemos, ahora la tabla tiene una nueva columna llamada "DNI" que se ha autoincrementado automáticamente para cada fila. El resultado de la consulta nos muestra esta nueva columna con los valores 1, 2 y 3 para cada fila respectivamente.

Los indicadores también nos ayudar a relacionar tablas. Imaginemos que en una tabla aparece también el indicador de otra tabla, por ejemplo, en una tabla de consultas médicas donde aparece el propio ID de cada consulta y el ID de la persona que asistió (DNI). Esto nos ayuda a identificar el nombre de la persona que asistió a cada consulta.

Para ver mejor estas relaciones vamos a usar la base de datos que aparece en el siguiente link: https://en.wikiversity.org/wiki/Database_Examples/Northwind/SQLite. Copiamos el código, creamos una nueva base de datos en DB Browser y pegamos el código en el ejecutor de SQL.

Ordenar y nombrar tablas

A la hora de visualizar tablas podemos pedir que en la visualización las columnas tengan otro nombre usando AS.

Para ordenar las filas segune el valor de una columna usamos ORDER BY Nombre Columna ASC/DESC

ASC implica ascendiente (y es el que viene por defecto, no hace falta que lo pongamos) y DESC descendiente (si queremos los valores menores arriba o abajo).

Este es un texto importante en cursiva y este es texto en cursiva simple.

SQL
SELECT ProductName as Nombre_Producto, Price as Precio FROM Products 
order by	Price DESC

En general el peso de cada formato es:

NULL (celdas vacias) → Números → Carácteres especiales → Letras

Si queremos que los NULL nos aparezcan al principio/final ponemos NULLS LAST/FIRST, después del ASC/DESC.

También podemos ordenar según varias columnas, siendo la primera la que manda más.

SQL
SELECT * FROM Products
ORDER BY CategoryID DESC, Price NULLS LAST;

Filtrar datos

Para filtrar datos usamos el WHERE y un operador comparativo.

SQL
SELECT ProductID, ProductName as Nombre_Producto, Price as Precio FROM Products 
WHERE ProductID = 14;

En este caso nos aparecerá el producto con ID 14. Para texto se tiene que poner entre ' ' y los operadores son <, >, <=, >=, =, !=.

Si queremos usar varios a la vez usamos AND y OR.

SQL
SELECT ProductID, ProductName as Nombre_Producto, Price as Precio FROM Products 
WHERE ProductID != 14 and Price > 30

En este caso nos aparecerán los productos con ID distinto a 14 y con un precio mayor a 30.

SQL
SELECT*FROM Employees
where FirstName ='Nancy' or FirstName = 'Anne'

Y si queremos el contrario de un conjunto usamos NOT.

SQL
SELECT * FROM Customers
where not Country='USA' AND NOT Country ='France'

En este caso nos aparecerán los clientes que no sean ni de USA ni de Francia.

La función BETWEEN nos permite nos sirve para buscar entre dos valores.

SQL
SELECT * FROM Customers
WHERE CustomerID BETWEEN  50 AND 55
SQL
 SELECT * FROM Employees where BirthDate BETWEEN '1950-0-1' and '1960-0-1'

Si queremos limitar el número de resultados de la búsqueda podemos usar LIMIT al final del código.

SQL
SELECT * FROM Customers
WHERE CustomerID > 50 AND NOT Country='USA'
LIMIT 5

La función Random nos permite seleccionar filas de forma aleatoria.

SQL
SELECT * FROM Products
WHERE NOT CategoryID = 6
AND not SupplierID = 1
and Price <= 30
order by random()
LIMIT 3

La función LIKE nos permite buscar patrones en los datos de texto. Por ejemplo, sabemos que el apellido de una persona empieza por F pero no sabemos como acaba

SQL
SELECT * FROM Employees
WHERE LastName LIKE 'F%';

O que contiene la r

SQL
SELECT * FROM Employees
WHERE LastName LIKE '%r%';

Si sabemos que empieza por la F y que tiene exactamente seis letras usamos "_"

SQL
SELECT * FROM Employees
WHERE LastName LIKE 'F_____;

Si estamos buscando varios valores, envez de usar el OR para cada valor, podemos usar IN y la lista de valores deseados.

SQL
SELECT * FROM Products
where SupplierID = 3
or SupplierID = 4
or SupplierID = 9
or SupplierID = 14
SQL
SELECT * FROM Products
WHERE SupplierID IN (3, 4, 9, 14);

Cuando estamos seleccionando las columnas que queremos visualizar también podemos pedir sus derivados:

SQL
SELECT Price, Price*1.21 as Precio_IVA FROM Products;

Dentro de los derivados también podemos usar funciones, como por ejemplo las funciones COUNT() para contar cuantos resultados nos devuelve, SUM() para la suma de todos sus valores o AVG() para la media de esa columna.

SQL
SELECT count(Price) as Cantidad_Precios, sum(price) as Suma_Precios, Round(AVG(price),2) as Precio_Medio FROM Products
--WHERE ProductName IS NOT NULL

Si queremos ver la suma de los valores, pero según una característica en concreto, usamos GROUP BY. Esto nos permite agrupar esta suma según dicha característica.

SQL
SELECT SupplierID, count(Price) as Cantidad_Precios, sum(price) as Suma_Precios, Round(AVG(price),2) as Precio_Medio FROM Products
WHERE Price IS NOT NULL
GROUP by SupplierID
ORDER by Precio_Medio DESC;

Si queremos filtrar usando una de las columnas que acabamos de calcular no podemos usar WHERE, tenemos que usar HAVING. Funciona de la misma manera per para columnas calculadas.

SQL
SELECT SupplierID, count(Price) as Cantidad_Precios, sum(price) as Suma_Precios, Round(AVG(price),2) as Precio_Medio FROM Products
WHERE Price IS NOT NULL
GROUP by SupplierID
HAVING Precio_Medio>40
ORDER by Precio_Medio DESC

Subconsultas

Aquí viene lo divertido, y complicado...

Recordemos que hemos dicho que los indicadores nos ayudaban a identificar tablas y relacionarlas con otras. Aquí es donde nos van a ser útiles.

Supongamos que queremos ver la cantidad de productos vendidos y su nombre para cada producto, pero tenemos las cantidades vendidas y los nombres en tablas distintas. Vamos a tener que "curzar" las tablas y vemos que ProductID aparece en ambas.

Vamos a asentarnos primero en una, OrderDetails por ejemplo, y queremos la suma de las cantidades por producto. Hasta ahí todo bien.

SQL
select ProductID,
	sum(Quantity) as Cantidad_total
FROM	OrderDetails
GROUP by ProductID

Ahora queremos el nombre. Para ello tenemos que hacer lo que se llama una subconsulta: buscamos una consulta que nos devuelva una tabla con una sola columna con los nombre de los productos:

SQL
SELECT ProductName FROM Products

Y los tenemos que hacer coincidir, así que usamos un WHERE Nombre de la tabla A.Nombre de la columna = Nombre de la tabla B.Nombre de la columna.

Las subconsultas son consultas anidadas dentro de otra consulta. Nos permiten realizar operaciones más complejas y obtener resultados que no podríamos obtener con una sola consulta.

SQL
select ProductID,
	sum(Quantity) as Cantidad_total, 
	(SELECT ProductName FROM Products
	WHERE Products.ProductID = OrderDetails.ProductID) as Name
FROM	OrderDetails
GROUP by ProductID

En este caso, la subconsulta (SELECT ProductName FROM Products WHERE Products.ProductID = OrderDetails.ProductID) se ejecuta para cada fila de la consulta principal. Para cada ProductID en OrderDetails, la subconsulta busca el ProductName correspondiente en la tabla Products y lo devuelve como una nueva columna llamada "Name".

SQL
SELECT 
    (SELECT ProductName FROM Products
		WHERE ProductID = OrderDetails.ProductID) as Nombre_prod,
	sum(Quantity) as Qty_total,
	(SELECT Price FROM Products
		WHERE ProductID = OrderDetails.ProductID)as Precio_prod,
	sum(Quantity)*(SELECT Price FROM Products
		WHERE ProductID = OrderDetails.ProductID) as Recaudación
FROM OrderDetails
GROUP BY ProductID
ORDER by Recaudación desc

Esto se puede hacer con varias tablas a la vez.

SQL
SELECT ContactName,
	(SELECT sum(quantity) from OrderDetails od, Orders o
	WHERE o.OrderID = od.OrderID AND c.CustomerID = o.CustomerID) as Cantidad_Comprada
from Customers c

GROUP by c.CustomerID
ORDER by Cantidad_Comprada DESC

JOINs

Los JOINs son una forma de combinar filas de dos o más tablas basándose en una columna relacionada entre ellas. Son fundamentales para trabajar con bases de datos relacionales, ya que nos permiten obtener información de varias tablas en una sola consulta.

Por ejemplo, si antes hemos hecho

SQL
SELECT * from Employees e, Orders o
where e.EmployeeID = o.EmployeeID

Esto es un JOIN implícito, pero también podemos hacerlo explícito usando la palabra JOIN:

SQL
SELECT * from Employees e JOIN Orders o
ON e.EmployeeID = o.EmployeeID

En este caso, estamos combinando las tablas Employees y Orders basándonos en la columna EmployeeID que es común a ambas tablas. El resultado de esta consulta será una nueva tabla que contiene todas las columnas de ambas tablas para las filas donde EmployeeID coincide.

Existen varios tipos de JOINs, siendo los más comunes:

  • INNER JOIN: Devuelve solo las filas que tienen coincidencias en ambas tablas (funciona también con solo JOIN).
  • LEFT JOIN: Devuelve todas las filas de la tabla izquierda y las filas coincidentes de la tabla derecha. Si no hay coincidencia, las columnas de la tabla derecha tendrán valores NULL. SELECT * from A LEFT JOIN B on A.id = B.id.
  • RIGHT JOIN: Devuelve todas las filas de la tabla derecha y las filas coincidentes de la tabla izquierda. Si no hay coincidencia, las columnas de la tabla izquierda tendrán valores NULL. SELECT * from B LEFT JOIN A on B.id = A.id.
  • FULL OUTER JOIN: Devuelve todas las filas cuando hay una coincidencia en una de las tablas. Si no hay coincidencia, las columnas de la tabla sin coincidencia tendrán valores NULL. SELECT * from A FULL JOIN B on A.id = B.id.
SQL
SELECT 
	FirstName, 
	e.LastName,
	sum(Quantity)as Uds_tot
FROM Employees e JOIN Orders o
on e.EmployeeID = o.EmployeeID 
JOIN OrderDetails od
on o.OrderID = od. OrderID
group by e.EmployeeID
ORDER by Uds_tot
SQL
SELECT 
	FirstName, 
	e.LastName,
	sum(Quantity)as Uds_tot
FROM Employees e LEFT JOIN Orders o
on e.EmployeeID = o.EmployeeID 
LEFT JOIN OrderDetails od
on o.OrderID = od. OrderID
group by e.EmployeeID
ORDER by Uds_tot

Vemos que ahora nos ha aparecido Adam, que no tiene ventas

Union

La función UNION nos permite combinar los resultados de dos o más consultas SELECT en una sola tabla de resultados,es decir, unen tablas de manera vertical. Es importante que las consultas que se combinan con UNION tengan el mismo número de columnas y que las columnas correspondientes tengan tipos de datos compatibles.

Tenemos el UNION y el UNION ALL. La diferencia entre ambos es que UNION elimina los registros duplicados del resultado final, mientras que UNION ALL incluye todos los registros, incluso los duplicados.

SQL
SELECT ProductID, ProductName, Price FROM Products
WHERE ProductID BETWEEN 18 and 20

UNION

SELECT ProductID, ProductName, Price FROM Products
WHERE ProductID BETWEEN 19 and 21

En este caso, el resultado de la consulta incluirá los productos con ID 18, 19, 20 y 21, pero el producto con ID 19 solo aparecerá una vez debido a que UNION elimina los duplicados.

SQL
SELECT ProductID, ProductName, Price FROM Products
WHERE ProductID BETWEEN 18 and 20

UNION ALL

SELECT ProductID, ProductName, Price FROM Products
WHERE ProductID BETWEEN 19 and 21

En este caso, el resultado de la consulta incluirá los productos con ID 18, 19, 20 y 21, y el producto con ID 19 aparecerá dos veces debido a que UNION ALL no elimina los duplicados.

Vistas

Una vista es una tabla virtual que se basa en el resultado de una consulta SQL. Las vistas no almacenan datos en sí mismas, sino que muestran datos de una o más tablas. Por ejemplo, para el ejercicio anterior que hemos visto podríamos hacerlo de la siguiente manera:

SQL
CREATE VIEW View_Uds_Tot as
SELECT
	FirstName,
	LastName,
	(SELECT sum(od.Quantity) from [Orders] o, [OrderDetails] od
	WHERE o.EmployeeID =e.EmployeeID and od.OrderID = o.OrderID) as Uds_tot
FROM [Employees] e
SQL
SELECT * from View_Uds_Tot
WHERE Uds_tot > (SELECT avg(Uds_tot) FROM View_Uds_Tot)

En este caso, hemos creado una vista llamada View_Uds_Tot que muestra el nombre y apellido de cada empleado junto con la cantidad total de unidades vendidas por cada uno. Luego, podemos consultar esta vista para obtener solo los empleados que han vendido más unidades que el promedio.

De esta manera consseguimos un código más limpio y fácil de entender, ya que no tenemos que repetir la subconsulta cada vez que queremos obtener la cantidad total de unidades vendidas por cada empleado.

Es importante que boremos las vistas cuando ya no las necesitemos

SQL
DROP VIEW View_Uds_Tot

Transacciones y prevención de desastres

Alterar una base de datos es un tanto peligroso, ya que si borras algo que no debes ya no hay marcha atrás. Para prevenir este tipo de fatalidades usamos las transacciones.

Una transacción te crea como una "burbuja" que no se aplica definitivamente hasta que tú lo decides.

SQL
BEGIN TRANSACTION;

UPDATE Employees 
SET FirstName = 'Eustaquia' 
WHERE FirstName = 'Nancy';

Supongamos que la hemos cagado y al final no teníamos que hacer esto. ROLLBACK te permite tirar para atrás deshaciendo todo lo que ha ocurrido.

SQL
ROLLBACK;

Si por el contrario, queremos que los cambios se apliquen definitivamente, usamos COMMIT.

SQL
COMMIT;
